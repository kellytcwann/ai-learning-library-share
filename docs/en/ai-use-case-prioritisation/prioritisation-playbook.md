# AI Use Case Prioritisation Playbook

A prioritisation playbook for turning AI ideas into a shortlist based on pain point, value, data readiness, controls, and delivery dependency.

## Step-by-Step Approach

1. Start from pain point: NIGO, missing documents, quote explanation, product rules search.
2. Define user and journey step: agent, sales support, operations, underwriter, product team.
3. Confirm data source: product guide, SOP, application data, document checklist, BRD.
4. Define AI support: search, summary, extraction, quality check, plain-language explanation.
5. Design controls: citations, human review, audit log, approved wording, exception handling.
6. Define metric: NIGO rate, cycle time, search time, manual rework, QA defects.
7. Bucket: quick start, controlled pilot, deep integration, defer.

## Discovery Questions

- Which pain point is this use case solving?
- What data would the AI use, and is it approved?
- Who reviews the output?
- Could it affect suitability, underwriting, pricing, or customer communication?
- Does it require POS integration?
- What metric would show improvement?

## Risk / Compliance Considerations

- Customer-facing language needs approval and version control.
- Product recommendation support needs suitability guardrails.
- Sensitive health/financial data needs access and retention controls.
- AI output used in sales or underwriting workflow should be logged.
- Automated decisioning should be treated as high risk unless governance is explicit.

## Expected Outputs

- Prioritised AI use case shortlist.
- Scoring rationale.
- Data and integration assumptions.
- Governance / compliance notes.
- Recommended next step for each use case.

## Meeting-Ready Wording

> “I suggest prioritising AI use cases by pain point, value, data readiness, control requirement, and POS dependency rather than by technology appeal.”
