# POS Assessment Playbook

A practical playbook for assessing a current POS / sales platform across journey, users, systems, data, product rules, vendor dependencies, controls, and AI opportunities.

## Objective

Assess the current POS / sales platform current state and translate pain points into evidence-based options and next-step recommendations.

## When To Use

- Before a POS discovery workshop.
- When preparing a current-state assessment approach.
- When turning POS pain points into recommendations.

## Step-by-Step Approach

1. Confirm scope: product line, channel, users, system boundary, expected output.
2. Map current-state journey: needs analysis, illustration, application, documents, submission, underwriting handoff.
3. Collect evidence: process maps, screenshots, NIGO reasons, case logs, sample BRDs, user feedback.
4. Identify pain point and root cause: process, system, data, vendor, or control issue.
5. Assess POS capabilities: quote, validation, document checklist, case tracking, reporting, configuration.
6. Shape options: optimise process, enhance platform, add AI assistant, improve integration, vendor change, longer-term platform change.

## Discovery Questions

| Topic | Questions |
|---|---|
| Scope | Which products, channels, and user groups are in scope? |
| Journey | Which steps are inside the current POS / sales platform and which are outside? |
| Pain point | Where do agents or operations teams spend the most time? |
| Data | Which data is structured, reused, or rekeyed? |
| Product rules | Which rules are configured, manual, or vendor-controlled? |
| Metrics | Do we have NIGO rate, STP rate, cycle time, drop-off, or rework volume? |

## Risk / Compliance Considerations

- Suitability and disclosure steps must be traceable.
- Customer, health, and financial data require access and retention controls.
- AI outputs should not bypass underwriting, compliance, or adviser review.
- Pain points should be supported by evidence, not only anecdotes.

## Expected Outputs

- Current-state journey map with system touchpoints.
- Pain point and root cause summary.
- POS capability gap table.
- Vendor and dependency assessment.
- AI opportunity list with governance notes.

## Meeting-Ready Wording

> “To make the POS assessment actionable, I suggest capturing each issue by journey step, user impact, root cause, evidence, dependency, and possible recommendation.”
