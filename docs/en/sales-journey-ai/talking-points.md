# Sales Journey AI Talking Points

Use these lines for professional discussions about Sales Journey AI opportunities.

## Positioning

> “Sales Journey AI should not start with automated recommendation or automated underwriting. We should start with lower-risk, agent-facing, human-reviewed support that reduces search effort, document errors, submission rework, and quote explanation effort.”

## Quick Wins

- “Agent knowledge assistant can use approved product guides, SOPs, and FAQs for grounded answers.”
- “Document checklist validation can help reduce missing documents and NIGO.”
- “Quote explanation support should use approved wording and adviser review.”
- “Pre-submission quality checks can reduce missing fields, inconsistent information, and rework.”
- “BRD rule extraction can support product/BA teams, but output needs source traceability and human sign-off.”

## Sensitive Use Cases

- “Product recommendation affects suitability and should not be treated as a simple quick win.”
- “Underwriting approve/reject decisions require clear human decision ownership.”
- “Customer-facing advice needs approved wording, disclaimer, and compliance review.”
- “Health and financial data require access, consent, retention, and audit controls.”

## Strong Close

> “I suggest prioritising by value, data readiness, human review, compliance risk, and POS dependency, rather than by how innovative the AI idea sounds.”
