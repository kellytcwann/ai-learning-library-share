# Sales Journey AI Identification Playbook

A practical playbook for identifying AI opportunities across the insurance sales journey while separating quick wins from sensitive use cases.

## Step-by-Step Approach

1. Map journey: agent preparation, FNA, product search, illustration, application, documents, submission, underwriting handoff, follow-up.
2. Identify pain points: slow search, missing documents, duplicate entry, hard-to-explain quotes, NIGO, unclear status.
3. Classify user: agent, sales support, operations, underwriter, product/BA team, customer.
4. Check source data: product guide, SOP, FAQ, BRD, application fields, document checklist, case status.
5. Define AI support: search, summarise, classify documents, check completeness, explain quote, extract rules.
6. Set controls: human review, approved sources, citations, audit trail, role-based access, compliance sign-off.
7. Split priority: quick win, controlled pilot, deep integration, sensitive/defer.

## Discovery Questions

- Which journey step has the most manual work or rework?
- What product/process information do agents search for most often?
- Can top NIGO reasons be reduced through pre-submission checks?
- Are there approved product/process materials for grounded answers?
- Which use cases need compliance / risk / legal sign-off?

## Risk / Compliance Considerations

- Agent-facing AI is easier to start than customer-facing AI, but still needs source control and logging.
- Product recommendation may affect suitability and should not be treated as a low-risk quick win.
- Underwriting approval/rejection should not be automated without explicit governance.
- Customer, health, and financial data require privacy, consent, and retention review.

## Expected Outputs

- Sales journey AI opportunity map.
- Low-risk quick win shortlist.
- Sensitive use case watchlist.
- Data / POS dependency assumptions.
- Governance and compliance questions log.

## Meeting-Ready Close

> “Let’s separate quick wins from sensitive use cases. Quick wins should start with agent-facing, approved-source, human-reviewed workflows; recommendation and underwriting use cases need suitability, privacy, audit trail, and compliance sign-off.”
