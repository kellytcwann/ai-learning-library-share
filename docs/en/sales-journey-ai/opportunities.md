# Sales Journey AI Opportunities in Insurance

Start with lower-risk, agent-facing AI use cases such as knowledge search, document checks, quote explanation, and pre-submission quality checks. Treat product recommendation and underwriting decisioning as sensitive use cases.

## Low-Risk Quick Wins

| Use Case | User | AI Support | Data Needed | Control |
|---|---|---|---|---|
| Agent knowledge assistant | Agent | Search product rules, process, FAQ | Approved product guides, SOP, FAQ | Citations, access control, usage log |
| Document checklist validator | Agent / operations | Check document completeness | Checklist, uploaded documents | Confidence threshold, exception queue |
| Quote explanation assistant | Agent | Explain illustration in plain language | Illustration output, approved wording | Template, disclaimer, agent review |
| Pre-submission quality check | Agent / sales support | Check missing fields and inconsistencies | Application fields, product rules, document status | User confirmation, audit trail |
| BRD rule extraction | Product / BA team | Extract product rules from old templates | Legacy BRDs/templates | Source traceability, human sign-off |

## Sensitive Use Cases

| Sensitive Area | Why Sensitive | Confirm First |
|---|---|---|
| Product recommendation | May affect suitability and regulated advice | Suitability guardrails, adviser review, approval |
| Suitability assessment | Sales compliance and customer needs | Compliance sign-off, audit trail |
| Underwriting decisioning | Risk acceptance, rejection, loading | Underwriting governance, human decision owner |
| Customer-facing advice | May be treated as advice | Approved wording, disclaimer, review process |
| Health / financial data use | Sensitive personal data | Access, retention, consent, privacy review |

## Questions To Ask

- Which journey pain point does this AI use case solve?
- Is it agent-facing, operations-facing, underwriting-facing, or customer-facing?
- What data does AI use, and is it an approved source?
- Who reviews the output, and is there an audit trail?
- What metric proves value: NIGO rate, cycle time, search time, or rework?

## Meeting-Ready Wording

> “I suggest starting with lower-risk, agent-facing AI use cases such as knowledge assistant, document checklist, quote explanation, and pre-submission quality check. Product recommendation and underwriting support need separate review for suitability, data privacy, audit trail, and human control.”
