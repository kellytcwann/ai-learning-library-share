# Sales Journey AI Notes

AI in the sales journey should be framed around user pain points, approved data, human review, controls, and measurable outcomes.

## Concept

Sales Journey AI supports agents, operations, underwriters, or product teams with search, summarisation, checks, explanation, and next-action support.

```text
Pain point → user → data → AI support → human control → metric
```

## Why It Matters

Insurance sales involves suitability, disclosures, customer data, health/financial information, and underwriting. AI can reduce manual work, but it needs review, audit trail, approved sources, and clear accountability.

## Examples

| Journey Step | AI Support | Control |
|---|---|---|
| Product query | Agent asks approved product/process questions | Approved sources + citations |
| Illustration | Explain quote outputs in plain language | Approved wording + disclaimer |
| Application | Check missing or inconsistent fields | Agent confirms before submission |
| Document upload | Identify missing KYC/supporting documents | Confidence threshold + exception queue |
| BRD reformat | Extract product rules from old templates | Source traceability + reviewer sign-off |

## Key Terms

| Term | Meaning |
|---|---|
| Grounding | AI answers are based on approved sources. |
| Human-in-the-loop | A person reviews important outputs or decisions. |
| Audit trail | Logs of output, source, user, and time. |
| Data readiness | Data is available, structured, accurate, and approved. |
| Suitability guardrail | Control preventing AI from bypassing sales suitability requirements. |

## Meeting-Ready Wording

> “I suggest assessing each AI idea by user pain point, data readiness, human review point, compliance risk, and measurable outcome before prioritising it.”
