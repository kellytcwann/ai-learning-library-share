# POS Assessment Playbook

這頁是 current POS / sales platform current-state assessment 的實戰方法，幫讀者把流程、系統、資料、vendor、產品規則和 AI 改善機會放在同一張分析地圖上。

## Objective

用結構化方式評估 target insurance client current POS / sales platform 的 current state：流程、用戶、系統、資料、產品規則、vendor 依賴、控制點，以及 AI / automation 改善機會。

## When To Use

- 準備 current POS / sales platform discovery workshop。
- 整理 current-state assessment approach。
- 需要把 POS pain points 轉成 recommendations。

## Step-by-Step Approach

1. **確認 scope**
   - product line、channel、user group、system boundary、expected output。
2. **畫 current-state journey**
   - needs analysis、illustration、application、document upload、submission、underwriting handoff。
3. **收集 evidence**
   - process map、screenshots、NIGO reasons、case logs、sample BRD、user feedback。
4. **找 pain point 和 root cause**
   - process issue、system limitation、data issue、vendor dependency、control gap。
5. **評估 POS capability**
   - quote、validation、document checklist、case tracking、reporting、configuration。
6. **整理 options**
   - optimize current process、enhance current POS / sales platform、add AI assistant、improve integration、vendor change、longer-term platform change。

## Discovery Questions

| Topic | Questions |
|---|---|
| Scope | Which products, channels, and user groups are in scope? |
| Journey | Which steps are inside current POS / sales platform and which are outside? |
| Pain point | Where do agents or operations teams spend the most time? |
| Data | Which data is structured, reused, or rekeyed? |
| Product rules | Which rules are configured, manual, or vendor-controlled? |
| Vendor | What is the change request model and release cycle? |
| Metrics | Do we have NIGO rate, STP rate, cycle time, drop-off, or rework volume? |

## Risk / Compliance Considerations

- Suitability and disclosure steps must be traceable.
- Customer, health, and financial data need access and retention controls.
- AI outputs should not bypass underwriting, compliance, or adviser review.
- Vendor limitations may affect roadmap feasibility and cost.
- Pain points should be backed by evidence, not only anecdote.

## Expected Outputs

- Current-state journey map with system touchpoints.
- Pain point / root cause summary.
- POS capability gap table.
- Vendor and dependency assessment.
- AI opportunity list with governance notes.
- Prioritised next-step recommendations.

## Meeting-Ready Wording

> 「為咗令 POS assessment actionable，我建議每個 issue 都記錄 journey step、user impact、root cause、evidence、dependency 同 possible recommendation。」
