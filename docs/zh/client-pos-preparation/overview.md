# Preparation Overview

這頁是整套 learning library 的準備總覽，幫讀者用 facts、assumptions、open questions 方式理解保險 POS / sales platform transformation 工作。

## How To Use This Page

先用這頁建立全局，再跳到 POS assessment、insurance POS basics 或 meeting / discussion outputs。

## 如何使用

| Section | 用途 |
|---|---|
| Insurance POS Basics | 建立 POS、sales journey、product configuration 基礎 |
| Sales Journey AI | 討論 AI opportunities、quick wins、sensitive use cases |
| Product BRD / Template Reformat | 準備 legacy template / product rule reformat work |
| Meeting / Discussion Outputs | 準備 discovery questions、talking points、alignment questions |

## 工作脈絡

| 類型 | 內容 |
|---|---|
| 已知事實 | 本 library 聚焦 insurance POS / sales platform、Sales Journey AI、Product BRD / template reformat。 |
| 工作假設 | Current POS / sales platform 可能支援 quotation、illustration、application、document upload、submission。 |
| 工作假設 | AI 應先定位為輔助 agent / operations / product team，而非直接取代保險決策。 |
| 待確認問題 | Target insurance client 的 market、產品線、channel、vendor scope、legacy templates/items 定義、export 格式。 |

## 三個工作 Lens

| Lens | 要留意 |
|---|---|
| BA | 流程、需求、業務規則、例外處理、控制點、資料欄位 |
| PM | scope、owner、timeline、dependency、decision gate、交付物 |
| FDE | data flow、system touchpoint、integration、prototype / automation feasibility |
