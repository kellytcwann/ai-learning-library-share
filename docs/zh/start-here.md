# Start Here

呢頁幫讀者按當前目標快速決定讀咩。唔需要由頭讀到尾，按情境揀材料就得。

## If I only have 10 minutes

**Recommended pages**

- [Learning Index](client-pos-preparation/learning-index.md)
- [Talking Points](meeting-outputs/talking-points.md)

**Focus**

- 先掌握已知事實、工作假設、待確認問題。
- 記住 BA + PM + FDE lens。

**Practice**

> 「我想先分清楚已知 facts、working assumptions 同 open questions，避免太早落 recommendation。」

## If I need to prepare for a project alignment discussion

**Recommended pages**

- [Project Alignment Questions](meeting-outputs/project-alignment-questions.md)
- [Client POS / Sales Platform Preparation](client-pos-preparation/index.md)

**Focus**

- 問清楚 priority、expected output、stakeholders、next milestone。
- 不要只問「我要做咩」，要問「咩 output 對 team 最有用」。

**Practice**

> 「下次 touchpoint 前，你覺得交 question list、current-state summary、AI use case view，定 BRD tracker draft 最有幫助？」

## If I need to understand insurance POS / sales platform

**Recommended pages**

- [Insurance POS Basics](insurance-pos-basics/pos-basics.md)
- [Product Configuration](insurance-pos-basics/product-configuration.md)

**Focus**

- 搞清楚 sales journey：FNA、illustration、application、document upload、submission。
- 留意 product rules 點樣影響 quote、NIGO、referral、POS configuration。

**Practice**

> 「今次 POS scope 係咪 cover quote、illustration、application、document upload 同 submission，定只係其中一部分？」

## If I need to discuss Sales Journey AI

**Recommended pages**

- [AI Opportunities in Insurance](sales-journey-ai/opportunities.md)
- [Sales Journey AI Talking Points](sales-journey-ai/talking-points.md)
- [Identification Playbook](sales-journey-ai/identification-playbook.md)

**Focus**

- 先講 lower-risk, agent-facing AI。
- Product recommendation、suitability、underwriting decisioning 唔好當 quick wins。

**Practice**

> 「我建議先由 approved-source、agent-facing、human-reviewed use cases 開始，例如 knowledge assistant、document checklist 同 pre-submission quality check。」

## If I need to prepare BRD / template reformat work

**Recommended pages**

- [BRD Reformat Playbook](product-brd-template-reformat/brd-reformat-playbook.md)
- [Product Configuration](insurance-pos-basics/product-configuration.md)

**Focus**

- 先確認 a batch of legacy templates/items 係咩：products、templates、documents、rules，定 export rows。
- 分清楚 documentation clean-up 同 system configuration / export mapping。

**Practice**

> 「Before reformatting，想先 confirm count unit、target export format、required fields 同 review owner。」

## If I need meeting-ready talking points

**Recommended pages**

- [Talking Points](meeting-outputs/talking-points.md)
- [Discovery Questions](meeting-outputs/discovery-questions.md)
- [Sales Journey AI Talking Points](sales-journey-ai/talking-points.md)

**Focus**

- 用香港 team 自然講法，保留必要英文專業詞。
- 每句都盡量連到 scope、pain point、evidence、owner、dependency 或 control。

**Practice**

> 「為咗令 discussion actionable，我建議每個 pain point 都 capture journey step、owner、evidence、system touchpoint、control impact 同 next action。」
