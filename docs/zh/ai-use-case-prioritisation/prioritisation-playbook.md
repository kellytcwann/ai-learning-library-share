# AI Use Case Prioritisation Playbook

這頁幫讀者把 AI ideas 排成可交付的 shortlist，避免只停留在概念層。

## Objective

幫讀者把 Client POS / Sales Platform sales journey AI ideas 排優先次序，確保每個 use case 都有清楚 pain point、data、control、metric 和 delivery dependency。

## When To Use

- Team 開始討論「AI 可以放在哪裡」。
- 需要把 AI ideas 變成 shortlist。
- 需要準備 use case matrix、roadmap 或 pilot recommendation。

## Step-by-Step Approach

1. **從 pain point 開始**
   - 例如 NIGO、文件錯漏、quote 難解釋、產品規則難找。
2. **定義 user 和 journey step**
   - agent、sales support、operations、underwriter、product team。
3. **確認 data source**
   - approved product guide、SOP、application data、document checklist、BRD。
4. **定義 AI support**
   - search、summary、field extraction、quality check、plain-language explanation。
5. **設計 control**
   - citations、human review、audit log、approved wording、exception handling。
6. **定義 metric**
   - NIGO rate、cycle time、search time、manual rework、QA defects。
7. **分桶**
   - quick start、controlled pilot、deep integration、defer。

## Discovery Questions

- Which pain point is this use case solving?
- What data would the AI use, and is it approved?
- Who reviews the output?
- Could it affect suitability, underwriting, pricing, or customer communication?
- Does it require POS integration?
- What metric would show improvement?

## Risk / Compliance Considerations

- Customer-facing language needs approval and version control.
- Product recommendation support needs suitability guardrails.
- Sensitive health/financial data needs access and retention controls.
- AI output used in sales or underwriting workflow should be logged.
- Automated decisioning should be treated as high-risk unless governance is explicit.

## Expected Outputs

- Prioritised AI use case shortlist.
- Scoring rationale.
- Data and integration assumptions.
- Governance / compliance notes.
- Recommended next step for each use case.

## Meeting-Ready Wording

> 「我建議 AI use case priority 唔好只睇 technology appeal，而係睇 pain point、value、data readiness、control requirement 同 POS dependency。」
