# AI Use Case Matrix

這頁是可直接拿去討論的 AI use case matrix，用來比較 user、pain point、data、control、metric 和 dependency。

## How To Use

用這張表把 AI idea 變成可討論的 consulting output。不要只列技術，要列 pain point、data、control 和 metric。

## Matrix

| Use Case | User | Pain Point | Data Needed | Control | Metric | Dependency |
|---|---|---|---|---|---|---|
| Agent knowledge assistant | Agent | Product/process rules hard to find | Approved product guides, SOPs, FAQs | Citations, access control, usage log | Search time, repeat questions | Low POS integration |
| Quote explanation support | Agent / customer | Illustration is hard to explain | Illustration output, approved wording | Approved language, disclaimer | Call time, customer understanding | Medium |
| Application quality checker | Agent / sales support | Errors found after submission | Application fields, product rules | User confirms fixes | NIGO rate, defect rate | High POS dependency |
| Document checklist validator | Agent / operations | Missing documents delay cases | Checklist, uploaded documents | Confidence threshold, exception queue | Pending requirements, cycle time | Medium |
| BRD rule extraction | Product / BA team | Old templates inconsistent | Legacy BRDs/templates | Source traceability, reviewer sign-off | Reformat throughput, QA defects | Low |

## Prioritisation Questions

- What business metric improves?
- Is the data available and approved?
- Who reviews the AI output?
- Could this affect suitability, underwriting, pricing, or customer communication?
- Can it start outside current POS / sales platform, or does it need integration?
