    Discovery Questions¶

    這頁是按 stakeholder 整理的 discovery questions，語氣貼近香港 team / client 對話方式，同時保留必要英文專業詞。

    先問清楚 Scope¶

    • 「今次 target insurance client scope 係邊個 market、channel 同 product line？」
    • 「今次講嘅 current POS / sales platform 包括邊啲功能？係 quote、illustration、application、document upload、submission 都包，定只係其中一部分？」
    • 「呢個 assessment 最後要支持咩 decision？係 current-state diagnostic、vendor review、roadmap，定 implementation prep？」
    • 「我哋今次係 expected to assess、recommend、prototype，定 prepare implementation？」
    • 「之前提到嘅 a batch of legacy templates/items 係咩單位？係 products、templates、documents、rules，定 export rows？」
    • 「Export 係指 export 去 Excel、CSV、BRD template、system import file，定 consolidated tracker？」

    問 Business / Product Owner¶

    • 「邊啲 product rules 最常改？」
    • 「邊啲 rules 最容易有 ambiguity 或者要靠人手 interpret？」
    • 「Product rule change 通常邊個 approve？Product、compliance、underwriting，定 IT？」
    • 「而家 BRD 主要係 documentation 用，定會直接影響 POS configuration？」
    • 「如果 BRD wording 同 system behavior 唔一致，通常點處理？」

    問 Sales / Channel¶

    • 「Agent 喺 sales journey 邊一步最花時間？」
    • 「客戶通常喺邊啲位最容易唔明？Quote、illustration、benefits、exclusions，定 application requirements？」
    • 「current POS / sales platform 有冇邊啲位令 agent 要用 workaround，例如 Excel、email、manual note？」
    • 「如果可以改善一個 current POS / sales platform step，sales team 最想改邊個？」
    • 「有冇常見 case 係 agent 明明覺得應該 submit 到，但 system block 咗或者之後變 NIGO？」

    問 Operations / Underwriting¶

    • 「Top NIGO reasons 係咩？」
    • 「邊啲 supporting documents 最常 missing 或者 upload 錯？」
    • 「邊啲 validation 如果可以早啲做，就可以減少 rework？」
    • 「Underwriting handoff 時，通常缺少咩 case information？」
    • 「Referral case 最常見原因係 product rule、medical、financial、compliance，定資料唔完整？」

    問 IT / Current POS / Sales Platform / Vendor¶

    • 「邊啲 rules 可以喺 current POS / sales platform configure？邊啲要 vendor support？」
    • 「Product change / rule change 嘅 release cycle 通常幾耐？」
    • 「current POS / sales platform 同 CRM、underwriting、policy admin、payment、document system 有咩 integration？」
    • 「Journey data、case data、document status 可唔可以 extract 出嚟做 reporting 或 AI use case？」
    • 「而家有咩 audit log、role-based access、data retention controls？」
    • 「Vendor 有冇現成 AI feature、API、integration hook，定要另外 build？」

    問 Compliance / Risk¶

    • 「Sales journey 邊啲 step 一定要 suitability、disclosure、consent 或 approval evidence？」
    • 「邊類 AI use case 需要 compliance / legal / risk review？」
    • 「Customer、health、financial data 有咩使用限制？」
    • 「如果 AI support agent 或 operations work，要 log 啲咩？」
    • 「AI output 可唔可以直接俾 customer 睇？定一定要 agent review？」

    Meeting-Ready Close¶

    「為咗之後 assessment 可以 actionable，我建議每個 pain point 都 capture：journey step、owner、evidence、system touchpoint、control impact 同 potential next action。」
